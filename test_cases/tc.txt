Test1: Успешная оплата заказа с прохождением 3-D Secure

ID: Test1
Название: Успешная оплата заказа с прохождением 3-D Secure
Предусловия	Пользователь авторизован, выбран заказ, открыта страница оплаты. Используется тестовая карта, поддерживающая 3-D Secure (например, 3DS-имитация). Баланс карты достаточен. Платежный шлюз настроен на возврат успешного ответа.
Шаги:
1. Ввести номер тестовой карты.
2. Ввести срок действия (будущий месяц/год).
3. Ввести CVV.
4. Нажать кнопку «Оплатить».
5. При перенаправлении на страницу 3-D Secure ввести корректный код подтверждения.
6. Дождаться возврата на сайт магазина.
Ожидаемый результат:
1. Отображается сообщение об успешной оплате.
2. Статус заказа в личном кабинете изменён на «Оплачен».
3. На email пользователя приходит чек с деталями оплаты.

Test2: Оплата отклонена банком (недостаточно средств)

ID: Test2
Название: Оплата отклонена банком из-за недостатка средств
Предусловия	Пользователь авторизован, открыта страница оплаты. Используется тестовая карта, имитирующая отказ по причине «недостаточно средств». Платежный шлюз настроен на возврат ответа «Отклонено банком».
Шаги:
1. Ввести номер тестовой карты.
2. Ввести корректные срок действия и CVV.
3. Нажать «Оплатить».
4. При необходимости пройти 3-D Secure.
Ожидаемый результат:
1. Отображается сообщение об отказе: «Платёж отклонён банком. Попробуйте другую карту».
2. Статус заказа не изменяется (остаётся «Ожидает оплаты»).
3. Чек на email не отправляется.
4. Пользователь может повторить попытку с другой картой.

Test3: Ошибка платёжного сервиса при оплате

ID: Test3

Test4: Граничные значения срока действия карты (истекший срок)

ID: Test4
Название	Оплата картой с истекшим сроком действия
Предусловия	Пользователь авторизован, открыта страница оплаты. Используется тестовая карта, у которой месяц/год меньше текущей даты (например, прошлый месяц текущего года).
Шаги:
1. Ввести номер тестовой карты.
2. Ввести истекший месяц и год.
3. Ввести CVV.
4. Нажать «Оплатить».
Ожидаемый результат:
1. Появляется сообщение об ошибке: «Срок действия карты истек» или «Неверный срок действия».
2. Запрос к платежному шлюзу не отправляется (или шлюз возвращает ошибку, обрабатываемую как отказ).
3. Статус заказа не меняется.

Test5: Проверка клиентской валидации поля «Номер карты»

ID: Test5
Название: Валидация поля «Номер карты» при вводе некорректных символов
Предусловия	Пользователь авторизован, открыта страница оплаты.
Шаги:
1. В поле «Номер карты» ввести значение, содержащее буквы (например, «abcd»).
2. Перейти к следующему полю или попытаться нажать «Оплатить».
3. Повторить с номером карты менее 13 цифр и более 19 цифр.
Ожидаемый результат:
1. При вводе букв появляется сообщение: «Номер карты должен содержать только цифры».
2. При недостаточной/избыточной длине – сообщение о неверном формате.
3. Кнопка «Оплатить» неактивна или при нажатии форма не отправляется.

Test6: Проверка интеграции с платёжным шлюзом (обработка успешного ответа)

ID: Test6
Название: Проверка корректной обработки успешного ответа от платёжного шлюза
Предусловия	Пользователь авторизован, открыта страница оплаты. Есть доступ к базе данных, логам приложения и тестовому почтовому ящику. Платежный шлюз настроен на возврат статуса «Успешно» с корректными параметрами (ID транзакции).
Шаги:
1. Выполнить успешную оплату.
2. Проверить в базе данных статус заказа и сохранённый ID транзакции от шлюза.
3. Проверить логи приложения на наличие записи об успешной транзакции.
4. Проверить тестовый почтовый ящик на наличие чека.
Ожидаемый результат:
1. Статус заказа в БД изменён на «Оплачен».
2. В БД сохранён ID транзакции, полученный от шлюза.
3. В логах зафиксирован факт успешной оплаты.
4. На email отправлен чек с корректными данными заказа.

Задача с монеткой.
Вероятность события равна отношению числа благоприятных исходов к общему числу исходов:
Благоприятных исходов у нас 3, всего исходов 8. Вероятность = число благоприятных исходов / общее число исходов = 0.375
Ответ: 0.375
